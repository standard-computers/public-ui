Standards

Standards are the core of the Standard System. All data is saved, recorded, or logged using a Standard. If you save a file on your Standard System, there is a file Standard where all files are logged for faster search.

A Standard is a database table construct that sets the Standard for how data should represent an object. When creating a Standard, save the content with the file extension .stds. Each Standard requires its own file. The Standard's name and file name must match.

Starter Example

    
    vehicle: VHL {
        private vin string:36! VN *
        protected make string MK *
        protected model string MDL *
    }

A Standard should always start with the standard_name: STDREF { ... }. Standards will typically be referenced by the standard reference. Standard’s columns, variables, or properties are called constraints and go inside the brackets. Prepend a Standard's name with ! to enable change history.

Listing Standards

To list Standards, run stds or standards. To view a specific Standard, use stds or standards followed by the Standard name (not the Standard reference). Many web apps and client applications communicate with JSON. Append the standards command with json to view a Standard as JSON.
