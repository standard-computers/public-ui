The bulletin is how relay-connected Standard System(s) keep records updated with other systems. A bulletin is generated and sent every morning to the nodes connected to the relay.

Generation

Standard will only generate a bulletin if it is set to master-server mode (relay mode). You can set the bulletin generation cadence in the config.conf file.

Standard will check the history records to determine if any records have been created, updated, or deleted. The bulletin contains a list of record IDs that need updated by the connected Standard System(s). Connected systems begin downloading and updating the records.
